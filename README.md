Filter

Filter 是一个轻量级、隐私优先的安全过滤层，用于在文本或文档被提交给 AI 系统、翻译工具或在线平台之前，降低无意的敏感信息暴露风险。

产品目标

在不改变内容可读性的前提下，帮助用户生成更安全的输入版本。

Filter 的设计理念：

• 保持内容对人类与 AI 可理解
• 仅处理真正的敏感信息
• 不引入多余标记或提示文本
• 完全本地运行

核心保证

• 无需登录
• 不存储数据
• 不跟踪内容
• 所有处理均在浏览器本地完成
• 关闭或刷新页面即清除全部数据

为什么需要 Filter

现代 AI 系统、翻译工具与自动化平台通常具备强大的文本解析与结构识别能力。
用户在复制、粘贴或上传材料时，往往会无意中暴露不应被机器提取的敏感信息，例如：

• 联系方式
• 身份标识信息
• 财务数据
• 内部文档字段
• 可重新识别的个人或组织信息

Filter 的目标不是修改内容含义，而是：

✔ 在进入外部系统之前降低机器可解析的敏感暴露面

安全模型

Filter 主要关注：

✔ 防止机器 / AI / 自动化系统提取敏感信息

而不是：

• 限制人类视觉阅读
• 提供文档编辑功能
• 优化文件大小

在文档处理场景中，系统优先级：

安全性 > 可编辑性 > 文件大小

PDF 安全处理策略

文档输入采用 Raster Secure Model：

Document page → High-DPI rasterization → Opaque redaction → Rebuilt PDF

导出的文件特性：

• 不包含可提取文本层
• 不保留原始 PDF 对象结构
• 不包含隐藏字符串或文本对象
• 遮盖区域为完全不透明像素

这意味着：

✔ 敏感信息在结构层面不可恢复
✔ 不依赖阅读器或查看器的“遮盖机制”

不适用场景

Filter 并非为以下用途设计：

• 法律意义上的正式文档脱敏认证
• 对抗高强度取证级攻击
• OCR 重建 / 文档语义恢复
• 内容加密 / DRM 控制
• 人类视觉匿名化需求

如果你的目标是：

✔ 法律合规审计
✔ 企业级数据治理
✔ 强对抗安全模型

则需要更专业的文档安全解决方案。

隐私与数据处理原则

Filter 的基本原则：

• 不存储用户输入或输出
• 不建立用户身份关联
• 不上传内容至服务器
• 不记录文本内容或命中结果

所有处理均在浏览器内存中完成。
刷新或关闭页面即销毁全部数据。

设计边界

Filter 是一个风险降低工具（risk-reduction tool），而不是：

• 安全保证系统
• 数据合规认证工具
• 法律责任替代机制

它的作用是减少常见误暴露，而非消除全部风险。
